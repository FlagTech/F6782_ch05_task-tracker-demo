# 任務追蹤器範例（Python）

這是一個小型的命令列任務追蹤器，用於 Codex 子代理（subagent）示範。

此程式在本機執行，不需要伺服器、網路連線或第三方套件。

## 執行程式

```bash
python app.py
```

任務資料會儲存在 `app.py` 旁的 `tasks.json` 中，無論從哪個目錄執行指令都一樣。

## 執行測試

```bash
python -m unittest discover -s tests -v
```

## 建議的手動測試

依下列順序使用選單：

1. 新增任務：`Buy milk`
2. 新增任務：`Write report`
3. 列出任務（選項 `2`）；預期 `#1` 與 `#2` 皆為未完成。
4. 完成任務 `#2`（選項 `3`）；預期出現完成訊息。
5. 顯示統計（選項 `4`）；預期總數 2、未完成 1、已完成 1。
6. 再次列出任務；預期 `#1` 為未完成、`#2` 為已完成。
7. 儲存並離開（選項 `5`）。
8. 再次執行 `python app.py` 並列出任務；預期任務與狀態維持不變。

## 預期行為

- 新增兩個任務應產生兩個不同的任務 ID。
- 完成任務 id `2` 應完成第二個任務。
- 儲存並重新啟動後，任務清單應保持不變。
- 已完成與未完成的任務數量應與可見清單一致。

## 儲存與復原

新的儲存會使用標準的 JSON 物件格式 `{"tasks": [...]}`。為了相容性，載入器也接受舊版的純陣列格式。資料在載入或取代前都會經過驗證。若 `tasks.json` 損毀或含有無效的任務欄位，程式會回報其完整路徑並在不覆寫檔案的情況下退出。請將損毀的檔案移至他處，或修復其 JSON 後再重新啟動。

選項 `5` 會儲存並離開。若儲存失敗，程式會保持開啟並將變更保留在記憶體中，以便重試儲存。選項 `6` 不會儲存，且在有待處理變更時會要求確認。按 Ctrl+C 或遇到輸入結束（end-of-input）會乾淨地退出，且不會寫入未儲存的變更。

## 專案面向

使用 Codex 子代理從多個角度檢視此專案：

- CLI 工作流程行為
- 任務狀態管理
- 檔案持久化
- 測試與文件一致性
